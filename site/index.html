Intent-First Programming Language

Describe what an app should do. Let AI synthesize the code.

Application Intent Model (AIM) is a human-readable, machine-parseable format for feature design. One intent entry can be enough for simple features, and optional facets add precision when teams need strict schema, flow, contracts, or persona traceability.

How teams use it: select a registry package, materialize files into local /ail, synthesize code, then keep editing the local AIM sources and rebuild anytime.

1. Start Light

Author only <feature>.intent with summary + requirements for fast product iteration.

2. Add Precision

Inline or linked SCHEMA, FLOW, CONTRACT, PERSONA facets when deterministic synthesis matters.

3. Rebuild Locally

Registry fetch is sync only. Local /ail remains source of truth for edits, regeneration, and long-term maintenance.

What makes AIM different

  • Natural-language intent is first-class, not comments around code.
  • Deterministic header: AIM: feature#facet@x.y.
  • Structured traceability from user-facing actions to system behavior.
  • Hybrid model supports both lightweight and high-fidelity specs.

Minimal Example

AIM: weather#intent@1.4

INTENT WeatherLookup {
  SUMMARY: "Get current weather by city."
  REQUIREMENTS:
    - User enters a city and sees current conditions.
}

Then evolve to detailed facets only when needed.
